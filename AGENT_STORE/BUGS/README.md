# BUGS Backlog

Track reproducible defects here so QA, DEV, and UX can coordinate fixes. Keep details factual and actionable. Close items only after verification, then move the file to `BUGS/RESOLVED/`.

## Naming
`BUG-YYYYMMDD-short-slug.md`

## Template
### Summary
### Environment
### Steps to Reproduce
### Expected Results
### Actual Results
### Fix Plan
### Verification Notes
